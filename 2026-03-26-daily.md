## 📄 Top Hugging Face Papers (AI/Agents)

### 1. EVA: Efficient Reinforcement Learning for End-to-End Video Agent
- **Authors:** Yaolun Zhang, Ruohui Wang, Jiahao Wang, et al. (SenseTime Research)
- **URL:** https://huggingface.co/papers/2603.22918
- **Votes:** 29
- **Abstract:** EVA introduces an Efficient Reinforcement Learning framework for End-to-End Video Agent that enables planning-before-perception through iterative summary–plan–action–reflection reasoning. The agent autonomously decides what to watch, when to watch, and how to watch, achieving query-driven and efficient video understanding. Uses a three-stage learning pipeline (SFT, KTO, GRPO) and achieves 6–12% improvement over general MLLM baselines on six video understanding benchmarks.

### 2. T-MAP: Red-Teaming LLM Agents with Trajectory-aware Evolutionary Search
- **Authors:** Hyomin Lee, Sangwoo Park, Yumin Choi, et al. (KAIST AI)
- **URL:** https://huggingface.co/papers/2603.22341
- **Votes:** 27
- **Abstract:** T-MAP addresses agent-specific vulnerabilities that emerge through multi-step tool execution, particularly in Model Context Protocol (MCP) ecosystems. The trajectory-aware evolutionary search method leverages execution trajectories to guide adversarial prompt discovery. Outperforms baselines in attack realization rate and remains effective against frontier models including GPT-5.2, Gemini-3-Pro, Qwen3.5, and GLM-5.

### 3. UI-Voyager: A Self-Evolving GUI Agent Learning via Failed Experience
- **Authors:** Zichuan Lin, Feiyu Liu, Yijun Yang, et al. (Tencent Hunyuan)
- **URL:** https://huggingface.co/papers/2603.24533
- **Votes:** 25
- **Abstract:** UI-Voyager is a two-stage self-evolving mobile GUI agent that addresses inefficient learning from failed trajectories and ambiguous credit assignment. Stage 1 employs Rejection Fine-Tuning (RFT) for autonomous data-model co-evolution. Stage 2 introduces Group Relative Self-Distillation (GRSD) for dense step-level supervision. Achieves 81.0% Pass@1 on AndroidWorld, exceeding human-level performance with a 4B model.

### 4. CarePilot: A Multi-Agent Framework for Long-Horizon Computer Task Automation in Healthcare
- **Authors:** Akash Ghosh, Tajamul Ashraf, Rishu Kumar Singh, et al. (MBZUAI, IIT Patna)
- **URL:** https://huggingface.co/papers/2603.24157
- **Votes:** 8
- **Abstract:** CarePilot introduces a multi-agent framework based on the actor–critic paradigm for long-horizon computer task automation in healthcare. The Actor integrates tool grounding with dual-memory mechanisms (long-term and short-term experience), while the Critic evaluates actions and provides corrective feedback. Also introduces CareFlow, a high-quality human-annotated benchmark for complex medical software workflows. Outperforms baselines by ~15.26% on the benchmark.

## 🧠 Key Trends & Insights

### 1. **Agentic Video Understanding is Maturing**
EVA's planning-before-perception paradigm represents a significant shift from passive video processing to active, adaptive agent-based watching. The ability to autonomously decide what/when/how to watch with iterative reflection mirrors human attention mechanisms.

### 2. **Agent Security is Critical**
T-MAP highlights that agent vulnerabilities differ fundamentally from chat-based LLM risks. Multi-step tool execution creates new attack surfaces, especially in MCP ecosystems. Red-teaming must evolve beyond single-turn jailbreaks to trajectory-aware adversarial search.

### 3. **Self-Evolving Agents are Breaking Performance Barriers**
UI-Voyager's 81% success rate with a 4B model (exceeding human performance) demonstrates that self-evolving training paradigms (RFT + GRSD) can dramatically improve sample efficiency. Learning from failed experiences, not just successes, is key.

### 4. **Domain-Specific Agent Frameworks are Emerging**
CarePilot shows that generic agents struggle with specialized domains like healthcare. The actor-critic pattern with dual memory mechanisms provides a template for building robust, domain-aware agents for complex workflows.

### 5. **AI Coding Impact Remains Concentrated**
The Claude Code analysis suggests AI-generated code is still primarily experimental. The gap between AI coding capability and production adoption remains significant—most AI contributions don't reach widely-used projects.
