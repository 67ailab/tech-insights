## 🔥 Top Stories

### Jensen Huang: "AI Agents Will Use Software Tools, Not Replace Them"
Fresh from Nvidia's strong Q4 earnings beat, Jensen Huang pushed back on the narrative that AI agents will replace enterprise software — instead arguing they'll *enhance* existing tools. A notable framing shift from the "agents replace everything" hype cycle.

### NIST Seeking Public Comment on AI Agent Security (Deadline: March 9)
[HN Discussion](https://news.ycombinator.com/item?id=47131689) — NIST is gathering input on novel agent threats including prompt injection, behavioral hijacking, and cascade failures. Key question: how existing security frameworks (STRIDE, attack trees) need to adapt for autonomous agents. They're also exploring agent registration/tracking analogous to drone registration.

### "Claws Are Now a New Layer on Top of LLM Agents"
[HN Discussion](https://news.ycombinator.com/item?id=47096253) — Simon Willison covered the emerging "claws" concept — a meta-layer sitting above LLM agents that manages orchestration, permissions, and coordination. Generating significant debate about whether we're building too many abstraction layers.

## 🛠️ Tools & Frameworks

### Cord – Constitutional AI Enforcement Engine for Autonomous Agents
[Show HN](https://news.ycombinator.com/item?id=47133376) — Intercepts every proposed agent action before execution. Designed to prevent jailbreaks hidden in external content from executing through the agent pipeline. Addresses the "no enforcement layer" gap in most shipped agents.

### Boardroom MCP – Multi-Advisor Governance for AI Agents
[Show HN](https://news.ycombinator.com/item?id=47130209) — Instead of fixing agent judgment with bigger models, this uses multiple "advisor" agents to deliberate on ambiguous decisions, considering second-order effects before execution.

### Hyperterse 2.0 – Schema-First MCP Server Framework
[Show HN](https://news.ycombinator.com/item?id=47142975) — Declarative framework for building MCP servers directly on production databases. Addresses the common pain point of wiring reliable structured data access for production AI agents.

### AgentVoice – Find Where AI Agents Get Stuck with Your API
[Show HN](https://news.ycombinator.com/item?id=47148311) — Triangulates agent traces, logs, and API specs to identify friction points (undocumented constraints, case-sensitivity issues) that don't show up as traditional errors.

### AgentBouncr – Governance Layer for AI Agents
[Show HN](https://news.ycombinator.com/item?id=47087311) — Permission layer with JSON-based policy engine, append-only audit trail, injection detection, and kill switch. Supports rate limits and approval workflows.

### Hugging Face Skills – Agent Skill Framework (6k+ Stars)
Now supporting Cursor plugins (Feb 2026), Claude Code, OpenAI Codex, and Gemini CLI. Standardizing the "Agent Skill" format across platforms. Could become the de facto standard for agent capability packaging.

### Smolagents on AWS
AWS published a guide for building agentic AI with Hugging Face's smolagents library — lightweight Python framework for multi-model agent workflows on AWS infrastructure.

## 📄 Notable Papers

### "Agents of Chaos" — Red-Teaming Autonomous Agents in Live Environments
[arXiv 2602.20021](https://arxiv.org/abs/2602.20021) — Exploratory red-teaming study of autonomous LLM agents deployed in a live lab with persistent memory, email, Discord access, file systems, and shell execution. Essential reading for anyone deploying agents in production.

### "Anatomy of Agentic Memory" — Taxonomy of Memory Systems
[arXiv 2602.19320](https://arxiv.org/abs/2602.19320) — Brain-inspired trichotomy of implicit, explicit, and agentic memory. Extends analysis to multimodal settings (vision, audio, embodied interaction). Identifies evaluation and system limitations.

### "MagicAgent: Towards Generalized Agent Planning"
[arXiv 2602.19000](https://arxiv.org/abs/2602.19000) — Foundation models designed specifically for generalized agent planning. Addresses the problem that current models excel at isolated tasks but fail to generalize, and multi-task training suffers from gradient interference.

### "Implicit Intelligence: Evaluating Agents on What Users Don't Say"
[arXiv 2602.20424](https://arxiv.org/abs/2602.20424) — Framework testing whether AI agents can move beyond literal instruction-following to genuine goal fulfillment. Introduces "Agent-as-a-World" harness for scalable evaluation through LLM-simulated environments.

### "The 2025 AI Agent Index"
[arXiv 2602.17753](https://arxiv.org/abs/2602.17753) — Comprehensive documentation of technical and safety features across deployed agentic AI systems. Useful reference for benchmarking and governance.

### "Codified Context: Infrastructure for AI Agents in Complex Codebases"
[arXiv 2602.20478](https://arxiv.org/abs/2602.20478) — MCP-based architecture for agent specifications, constitution documents, and knowledge bases. Includes factory agents for bootstrapping the architecture in new projects.

### GLM-5: From Vibe Coding to Agentic Engineering
Discussed at Hugging Face Journal Club (Feb 19). Technical report on the shift from ad-hoc "vibe coding" with AI to structured agentic engineering practices.

This week's HN is dominated by **agent governance tooling** — Cord, AgentBouncr, Temper Labs, Boardroom MCP. The "Agents of Chaos" paper provides academic backing for these concerns with real red-teaming data.
